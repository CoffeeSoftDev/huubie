Cierre Automatizado con Reportes

Cierre automatico, reportes historicos, exportacion y notificaciones.

Alcance

  • ● Todo lo de Propuestas 1, 2 y 3
  • ● Cierre automatico programado (cron job)
  • ● Reportes historicos de cierres
  • ● Exportacion a PDF/Excel
  • ● Notificaciones por cierre pendiente
  • ● Panel de reportes con filtros avanzados

Fases de Implementacion

1

Cierre Manual Completo (Base)

Implementar Propuesta 2 como base funcional. Incluye cierre con validaciones, preview de montos, checklist de verificacion y ticket de turno. Esta fase es prerequisito para todas las siguientes.

2

Cierre Automatico

Flujo del Cron Job

  CRON JOB (cada dia a las 23:59)
  |
  v
  +---> Verificar turnos abiertos del dia
  |     |
  |     +---> SI hay turnos abiertos:
  |     |     +---> Cerrar turnos automaticamente
  |     |     +---> Registrar cierre como "automatico"
  |     |
  |     +---> NO hay turnos abiertos:
  |           +---> Continuar
  |
  +---> Ejecutar cierre diario
  |     +---> Consolidar totales
  |     +---> Vincular turnos y ordenes
  |     +---> Generar registro en daily_closure
  |
  +---> Enviar notificacion
        +---> Email al administrador
        +---> Log del sistema

Pseudo-codigo PHP

class CronDailyClose {

    public function execute() {
        $config = $this->getAutoCloseConfig();

        if (!$config['enabled']) return;

        $openShifts = $this->getOpenShifts(date('Y-m-d'));

        foreach ($openShifts as $shift) {
            $this->autoCloseShift($shift['id']);
        }

        $result = $this->executeDailyClosure(date('Y-m-d'));
        $this->sendNotification($result);
        $this->logExecution($result);
    }
}

Tabla de Configuracion

CREATE TABLE daily_close_config (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    enabled         TINYINT(1) DEFAULT 0,
    close_time      TIME DEFAULT '23:59:00',
    auto_close_shifts TINYINT(1) DEFAULT 1,
    notify_admin    TINYINT(1) DEFAULT 1,
    notify_email    VARCHAR(255) DEFAULT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

Listado de Cierres con Rango

SELECT
    dc.id,
    dc.close_date,
    dc.total_shifts,
    dc.total_cash,
    dc.total_card,
    dc.total_transfer,
    (dc.total_cash + dc.total_card + dc.total_transfer) AS gran_total,
    dc.close_type,
    u.name AS closed_by
FROM daily_closure dc
LEFT JOIN users u ON dc.user_id = u.id
WHERE dc.close_date BETWEEN :fi AND :ff
    AND dc.active = 1
ORDER BY dc.close_date DESC;

Totales Acumulados

SELECT
    COUNT(*) AS total_cierres,
    SUM(total_shifts) AS total_turnos,
    SUM(total_cash) AS acum_efectivo,
    SUM(total_card) AS acum_tarjeta,
    SUM(total_transfer) AS acum_transferencia,
    SUM(total_cash + total_card + total_transfer) AS acum_total
FROM daily_closure
WHERE close_date BETWEEN :fi AND :ff
    AND active = 1;

Ventas por Dia (Grafico Linea)

SELECT
    close_date,
    (total_cash + total_card + total_transfer) AS total_dia
FROM daily_closure
WHERE close_date BETWEEN :fi AND :ff
    AND active = 1
ORDER BY close_date ASC;

Distribucion Metodos de Pago (Grafico Pastel)

SELECT
    SUM(total_cash) AS efectivo,
    SUM(total_card) AS tarjeta,
    SUM(total_transfer) AS transferencia,
    ROUND(SUM(total_cash) / SUM(total_cash + total_card + total_transfer) * 100, 1) AS pct_efectivo,
    ROUND(SUM(total_card) / SUM(total_cash + total_card + total_transfer) * 100, 1) AS pct_tarjeta,
    ROUND(SUM(total_transfer) / SUM(total_cash + total_card + total_transfer) * 100, 1) AS pct_transferencia
FROM daily_closure
WHERE close_date BETWEEN :fi AND :ff
    AND active = 1;

Exportacion PDF/Excel

Permite al usuario exportar los reportes de cierres en formato PDF (para impresion y archivo) y Excel (para analisis contable). Usa los mismos filtros del panel de reportes.

Query para Exportacion Excel

SELECT
    dc.close_date AS 'Fecha',
    dc.total_shifts AS 'Turnos',
    dc.total_orders AS 'Pedidos',
    dc.total_cash AS 'Efectivo',
    dc.total_card AS 'Tarjeta',
    dc.total_transfer AS 'Transferencia',
    (dc.total_cash + dc.total_card + dc.total_transfer) AS 'Total',
    dc.close_type AS 'Tipo Cierre',
    u.name AS 'Cerrado por'
FROM daily_closure dc
LEFT JOIN users u ON dc.user_id = u.id
WHERE dc.close_date BETWEEN :fi AND :ff
    AND dc.active = 1
ORDER BY dc.close_date ASC;

Notificaciones

Evento Canal Mensaje
Cierre automatico exitoso Email Cierre diario completado - Total: $X
Cierre automatico fallido Email + Sistema Error en cierre automatico - Accion requerida
Turnos sin cerrar (aviso) Sistema Hay X turnos abiertos pendientes de cierre
Reapertura de cierre Email + Log El cierre del dia X fue reabierto por [usuario]

Tabla de Notificaciones

CREATE TABLE daily_close_notifications (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    event_type      ENUM('auto_close_ok', 'auto_close_fail', 'shifts_pending', 'reopen'),
    channel         ENUM('email', 'system', 'both'),
    recipient_id    INT DEFAULT NULL,
    message         TEXT,
    is_read         TINYINT(1) DEFAULT 0,
    closure_id      INT DEFAULT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (closure_id) REFERENCES daily_closure(id)
);

Ventajas y Desventajas

Ventajas

  • ✓ Operacion completamente automatizada
  • ✓ Historial completo para auditoria y contabilidad
  • ✓ Exportable para analisis contable externo
  • ✓ Alertas proactivas ante problemas
  • ✓ Escalable para multiples sucursales

Desventajas

  • ✗ Implementacion extensa (9-12 semanas estimadas)
  • ✗ Requiere configuracion de cron jobs en servidor
  • ✗ Mas tablas en la base de datos
  • ✗ Mantenimiento significativo a largo plazo
  • ✗ Podria requerir modulo separado por complejidad
