Cierre Multi-turno con Dashboard

Dashboard operativo con estado del dia en tiempo real, graficos y comparativas.

Alcance

  • ● Todo lo de Propuestas 1 y 2
  • ● Dashboard visual con metricas en tiempo real
  • ● Graficos de ventas por turno y metodo de pago
  • ● Comparativa con dias anteriores
  • ● Timeline de actividad del dia
  • ● Ticket de cierre diario diferenciado

Query Comparativa

SELECT
    DATE(cs.opened_at) AS fecha,
    COUNT(cs.id) AS total_turnos,
    SUM(cs.total_cash) AS total_efectivo,
    SUM(cs.total_card) AS total_tarjeta,
    SUM(cs.total_transfer) AS total_transferencia,
    SUM(cs.total_cash + cs.total_card + cs.total_transfer) AS gran_total
FROM cash_shift cs
WHERE DATE(cs.opened_at) BETWEEN
    DATE_SUB(CURDATE(), INTERVAL 1 DAY)
    AND CURDATE()
GROUP BY DATE(cs.opened_at)
ORDER BY fecha ASC;

Query Timeline de Actividad

SELECT timestamp, event_type, description FROM (
    SELECT
        opened_at AS timestamp,
        'shift_open' AS event_type,
        CONCAT('Turno abierto por ', user_name) AS description
    FROM cash_shift
    WHERE DATE(opened_at) = CURDATE()

    UNION ALL

    SELECT
        closed_at AS timestamp,
        'shift_close' AS event_type,
        CONCAT('Turno cerrado por ', user_name) AS description
    FROM cash_shift
    WHERE DATE(closed_at) = CURDATE()
    AND closed_at IS NOT NULL

    UNION ALL

    SELECT
        created_at AS timestamp,
        'order_created' AS event_type,
        CONCAT('Pedido #', folio, ' creado - $', total) AS description
    FROM orders
    WHERE DATE(created_at) = CURDATE()
) events
ORDER BY timestamp DESC;

Reabrir Cierre (Admin)

Queries de Reapertura

UPDATE daily_closure
SET active = 0,
    reopened_at = NOW(),
    reopened_by = :user_id,
    reopen_reason = :reason
WHERE id = :closure_id;
UPDATE orders
SET daily_closure_id = NULL
WHERE daily_closure_id = :closure_id;
UPDATE cash_shift
SET daily_closure_id = NULL
WHERE daily_closure_id = :closure_id;

Validaciones

  • ● Solo usuarios con rol == 1 (administrador) pueden reabrir
  • ● Requiere motivo obligatorio de reapertura
  • ● Log de auditoria con usuario, fecha, hora y motivo

  • Timeline muestra eventos en orden cronologico

  • Ticket diario se genera e imprime correctamente

  • Reapertura requiere motivo y genera log

Ventajas y Desventajas

Ventajas

  • ✓ Vista completa del estado del dia en un solo lugar

  • ✗ Mas codigo para mantener a largo plazo
  • ✗ Posible over-engineering para operaciones pequenas
